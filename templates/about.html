How to use

To use this website you need to submit a PDB-file of a protein and a MOL2-file of a ligand. You can obtain the protein file by downloading it from the Protein Data Bank as PDB-file and the ligand MOL2-file can be obtained from the ZINC database. What each of the files should look like is described in detail in here:

  • PDB
  • MOL2

You can also specify the number of binding poses and the RMSD-cutoff. The number of binding poses is the number of times there is attempted to dock. So if you lower the number the website will run faster but the output will be worse. The number of binding poses is limited to 20, because anything higher will take too long and is in a vast majority of cases unnecessary.

RMSD or Root Mean Square Division is the measure of average distance between atoms of molecules and atoms of ligand configurations. It is measured in Ångström (Å). Decreasing this value will give higher accuracy but the docking process will take longer and higher values will give lower accuracy and a faster docking process.

What does it do?

The website takes your input (PDB- and MOL2-file) and runs it through an algorithm to simplify your file (LePro). after that step it runs the new file through an program called LeDock, which contains an AI trained to dock ligands onto proteins. This can take a very long time depending on the number of binding poses specified. This is because this number is the amount of iterations the AI will go through. Each iteration the AI tries to dock the ligand onto the protein, and if the pose works the AI calculates the score, and then it repeats this process. Finally, the outcomes are put in a file in decreasing order based on score and this webtool visualises the top 3 binding-poses using PLIP.

The history page

The history page is to retrace your steps. Every time you run the webtool the output gets saved to the history page. So if you accidentally delete one of the output files you can download it once again there.
