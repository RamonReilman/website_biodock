How to use

To use this website you need to submit a PDB-file of a protein and a MOL2-file of a ligand. You can obtain the protein file by downloading it from the Protein Data Bank as PDB-file and the ligand MOL2-file can be obtained from the ZINC database. What each of the files should look like is described in detail in here:

  • PDB
  • MOL2


You can specify the number of binding poses and the RMSD-cutoff.

The number of binding poses is the number of times the program attempts to dock the ligand onto the protein, so it's the number of iterations. If you lower the number, the webtool will run faster but the output will show less docking poses. The number of binding poses is limited to a maximum of 20, because any number higher will take too long and is in a vast majority of cases unnecessary.

RMSD or Root Mean Square Division is the measure of average distance between atoms of molecules and atoms of ligand configurations. It is measured in Ångström (Å). Decreasing this value will give higher accuracy but the docking process will take longer, higher RMSD-values will bundle similar docking poses together into 1, which gives lower accuracy and a faster docking process.

As output you will get a .dok-file that houses all of the calculated binding poses and images of all found binding poses. The images will look something like this:

What does it do?

The website takes your input (PDB- and MOL2-file) and runs it through an algorithm to simplify your file (LePro). Next it runs the result from Lepro through a program called LeDock, which contains an AI trained to dock ligands onto proteins, all found binding poses get saved and get a score assigned. Finally, the outcomes are put in a .dok-file in decreasing order based on score and this webtool visualises this data into images using PLIP (Protein-Ligand Interaction Profiler).

The history page

The history page is to retrace your steps. Every time you run the webtool the output gets saved to the history page. So if you accidentally delete one of the output files you can download it once again there.

Images explained

The website generates images of the ligand docked onto the protein. The molecules and interactions are all represented by different colors and are explained in a legend after using the webtool.

The imgs all come with a score based off of how likely that certain docking pose is. The lower the score, the more energy (kcal/mol) it gives and the more likely it is. The cluster number stands for the docking pose and it consists of at minimum 1 pose. With a high RMSD value poses that are similar get clustered together into one docking pose.

  • Example: Cluster 1 of poses: 5

The example respresents the first and most likely docking pose, it consists of the average result of the 5 poses combined together.
